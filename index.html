PSIDomainCheck

Check for malicious domains while preserving your privacy using Private Set Intersection (a cryptographic technique inside Secure Multi-party Computation family) and K-anonimity to strike a balance between perfomance, network bandwith and security

Private Set Intersection

Using private set intersection in a client-server configuration allows PSIDomainCheck to query for malicious domains without exposing the customer nagigation preferences and thus protection its privacy while maintaining the security benefits of using a ciberintelligence provider.

Image credit: TowardsDataScience.com - source

K-anonimity: Balance performance & security

Sharing the entire intel with the customer would require too many resources (Network, CPU, etc) so we combine PSI with K-anonimity to find the right balance between performance/usability and security/privacy.

Architecture

PSIDomainCheck is built on top of Libsodium for cryptography operations and FastAPI, Traefik and MongoDBfor its backend.
